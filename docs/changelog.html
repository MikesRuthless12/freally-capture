0.55.0 · in dev Jul 2, 2026 · Audio mixer + filters (audio half)

Audio mixer + filters — landed on main

The owned audio engine is in (the first half of 0.55.0; recording completes and publishes the milestone). Everything runs on the machine as owned classic DSP — no AI/ML anywhere. This work is on main now; the next downloadable installer ships when recording lands.

Highlights

  • Add an Audio Input Capture (microphone / line-in) or Audio Output Capture (desktop / system audio) — honest per OS: Windows WASAPI loopback, Linux monitor devices, macOS a virtual device (never silently installed).
  • A mixer with per-source volume, mute, and monitoring, assignable across up to 6 tracks, with a click-free fader and a per-source A/V sync offset.
  • Owned classic-DSP filters — spectral denoise (no ML), noise gate, compressor, limiter, 3-band EQ, gain, and sidechain ducking — editable live without clicks.
  • Monitoring to any output device, sidechain ducking, push-to-talk / push-to-mute global hotkeys, and a broadcast-standard LUFS loudness meter with live VU metering.
  • Captured audio never leaves your machine — it goes only to the mixer, the monitor device you pick, and (next milestone) the recording.

Multi-track recording with hardware encoders + the owned lossless freally-video codec completes 0.55.0 and publishes the next build.

0.40.0 Jul 2, 2026 · Compositor + scenes/sources
Download v0.40.0

0.40.0 — Compositor + scenes/sources (done)

The heart of the studio is in. Every scene composes on the owned wgpu GPU compositor — per-item move/scale/rotate/crop, seven blend modes, sustained 60 fps at 1080p — driven by the owned scene/source model: scenes share sources, items stack by z-order, and the whole collection autosaves locally and restores on launch. Each item carries an ordered on-GPU filter chain: chroma key (with spill suppression), color correction, LUT (.cube), blur, image mask, sharpen, scroll, and crop, all live-editable.

Highlights

  • Scenes and Sources rails work end to end: create/rename/reorder scenes; add displays, windows, webcams, images, colors, and text; toggle visibility, lock, and stack order; share one source across scenes.
  • Pixel-accurate on-canvas editing — drag, scale from corners or edges, rotate (with 15° snapping), and Alt-drag any edge to crop, straight on the program preview.
  • The Text source ships the complete Noto Sans family in the app (upright + italic + Arabic + Hebrew, every weight) with real shaping and RTL — text renders identically on every machine.
  • Honest per-source status (starting / live / error) and honest limits: a machine with no usable GPU says so instead of freezing.
  • Everything stays local — scene collections are plain JSON on your disk; frames never touch a socket.

This is an early development build: scenes, capture, and the compositor work. The audio mixer has since landed on main (above); recording and streaming arrive in the milestones below.

0.25.0 Jul 1, 2026 · Capture core
Download v0.25.0

0.25.0 — Capture core (done)

Real capture is in. Screens, windows, and webcams stream live into the program preview on all three OSes: Windows via DXGI Desktop Duplication (displays, with the mouse pointer blended in) and Windows.Graphics.Capture (windows); macOS via ScreenCaptureKit (12.3+), with the Screen-Recording permission requested honestly and a Settings deep-link on denial; Linux via the ScreenCast portal on Wayland — the system dialog picks the screen or window, the only capture Wayland allows — plus a direct X11 path. Webcams and capture cards enumerate with per-device formats and stream live. The preview pipe is in-process end to end: frames never touch a socket or disk.

Highlights

  • Display Capture, Window Capture, and Video Capture Device pickers in the Sources rail — the selected source previews live with a resolution / fps / dropped-frames overlay.
  • Per-OS capture behind one owned interface, with an honest dropped-frame count and permission flows that never pretend (macOS deep-links to the right Privacy pane; the Wayland picker explains portal-only capture).
  • Windows mouse-pointer blending in display capture; window capture survives resizes and display-mode switches.
  • Everything stays local — no accounts, no telemetry, zero outbound network calls.

This is an early development build: capture and the live preview work; the GPU compositor, audio mixer, recording, and streaming arrive in the milestones below.

0.10.0 Jul 1, 2026 · Foundation & repo

0.10.0 — Foundation & repo (done)

The cross-platform studio shell is built: a Tauri v2 + React (Havoc dark) UI over the owned Rust workspace (capture/compositor/scene/audio/encode/stream crates), the typed UI↔core bridge with a live stats feed, a persisted settings store, the 3-OS CI matrix, per-OS packaging (the Windows build is a GUI app with no console window), and this site.

Planned for the first public build (0.70.0 — Studio MVP)

  • Build scenes from sources on the owned wgpu GPU compositor, with per-source filters and transitions.
  • Multi-track recording via hardware encoders (NVENC/Quick Sync/AMF/VAAPI/VideoToolbox) + x264 fallback, plus the owned lossless freally-video codec.
  • Single-target live streaming (RTMP/RTMPS) to Twitch/YouTube/Kick/Facebook/Trovo/custom, with auto-reconnect and stream delay.
  • Studio Mode (preview/program) + transitions, a virtual camera, global hotkeys, profiles + scene collections, and a live stats dock.
  • No account, no telemetry, no cloud — streams go direct to the platform.

The detailed build plan is maintained privately; the public ladder lives in the roadmap below.
