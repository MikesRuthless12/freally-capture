1.1.0 — Deep Capture & Web (Phase 10, first slice)

Phase 10 (deep capture & web) opens. This cut ships the one complete, standalone piece — auto-export a shareable MP4 on stop — the roadmap feature owed since Build 12. The browser-source and virtual-camera runtimes are signed/component sub-milestones that land in later releases; nothing user-facing is exposed for them yet. Off by default, local, no network call.

  • Auto-export a shareable MP4 on stop — one toggle in Settings → Output (“Also save a shareable MP4 next to the master”), shown only when recording to the owned .frec and off by default. When a recording finalizes, a background job writes a double-click-playable MP4 copy beside the lossless master — visible in the Recordings background queue — through the same encoder as the manual export, with collision-safe naming. The master is never touched, and the copy is queued before any post-record pipeline so a Move step can’t relocate the master out from under it. Keep the crash-salvageable master and get a shareable file for free.

1.0.0 — Stable: stream-production overlays & launch polish

The 1.0.0 stable. The counted 100-feature roadmap (26 must-haves + 74 nice-to-haves) was completed at 0.900.0; this release ships the five launch-polish features parked for 1.0.0 — a pre-show countdown slate, a social bar, a featured chat banner, modal depth-of-field, and scene-history hygiene — plus a round of playlist/audio fixes. Everything new ships off or unbound by default, stays local, and adds no network call.

  • "Starting Soon" countdown slate — one click from Add Source: a full-canvas countdown that counts down TO a wall-clock time picked on a 12-hour clock picker (hours, 5-minute steps, AM/PM). At zero it flashes and holds — you clear it and cut to live by hand, never an auto-switch. Backdrops: gradient, solid, a still image (contain-fit, centred, never cropped), or a GIF/video as the scene's looping backdrop — with optional background music picked in the same dialog. Viewers of the stream/recording see it too.
  • Background Music source — one or more audio files (mp3/wav/flac/…) as an invisible looping playlist with optional shuffle, heard in preview and captured to the recording/stream; adding it mutes a video backdrop's own audio so only one background sound ever plays.
  • Social & channels bar — a generated overlay listing your handles: bundled platform rows (YouTube, Twitch, Kick, X, Instagram, TikTok, Facebook, Discord) as brand-coloured badges, plus custom rows and an optional header.
  • Featured chat message banner — pick any recent chat line from your running chat overlays and pin it to the program as a styled bottom banner; the colours persist, the pin clears on restart (by design) or with one click.
  • Modal depth-of-field — every modal blurs and dims the studio behind it (theme-aware, honours reduced-motion); companion surfaces like the teleprompter stay unblurred so the operator can keep reading.
  • Scene history hygiene — a Recent Scenes list with one-click Clear History, plus Scene Collection → Clear History to bulk-delete junk collections (the active one is never touched).
  • Audio visualizer classic colours — an optional classic green→yellow→red broadcast ramp for the bars and VU meters (the oscilloscope goes phosphor green).
  • Record a rehearsal — starting a rehearsal offers to also record it to a normal video file, so a dry run can become an edit-and-upload master without going live.
  • Fixed: Media Playlist audio — playlists now get their own mixer strip (fade, mute, monitor); an audio-only playlist shows the current track's name on canvas, cleanly.

0.910.0 — Teleprompter rework

The prompter grows up. A focused rework of the teleprompter shipped in 0.800.0: pauses become real objects you can't fat-finger, the scroll can pace to a song's tempo, an optional pre-roll counts the talent in, the projector hands the main window back to the operator, and the script can read itself aloud or finish your words as you type. Everything new is local and off or neutral by default — the autocomplete dictionaries are plain local data files (lazy-loaded, one language at a time) and Read aloud speaks through the operating system's own voice engine; nothing is bundled and nothing is fetched.

  • Caesura pause chips — an inline pause written as -- (a beat) or --N (an N-second hold) renders in the script editor as a single atomic pill: the caret cannot land inside it, one Delete/Backspace removes the whole token (never a stray dash), and copy/paste carries the marker intact. A pause reads and edits as one object instead of four loose characters.
  • Start countdown — an optional 3-2-1 pre-roll before the scroll begins, shown on both the operator preview and the talent projector, so the reader gets a beat to draw breath before the text moves.
  • BPM mode — pace the scroll in beats per minute (50–250) instead of characters per second, for reading lyrics in time with music. Switch modes freely; a change re-paces the running scroll live.
  • Real transport icons — the projector and operator panel now carry proper SVG play / pause / stop / rewind / fast-forward / restart controls in place of the placeholder glyphs.
  • Projector frees the main window — opening the talent projector now closes the operator modal, so you can run your livestream from the main window while the talent reads. With a start countdown armed, the projector auto-rewinds to the top, counts down, and plays from the top on open — one action puts the reader at zero and rolls.
  • Read aloud — preview a script through the operating system's own speech synthesis: Windows SAPI 5 / OneCore, macOS AVSpeechSynthesis, and on Linux the Web Speech API or a spd-say / espeak-ng fallback where installed. No TTS engine is bundled; the OS voice is used as-is (see THIRD-PARTY.md).
  • Ghost-text autocomplete — light-grey word and phrase suggestions appear inline as you type and are accepted with Tab. Each of the 18 app languages has its own dictionary, lazy-loaded for the language you're writing in only. Dictionary sources and their CC-BY / permissive licenses (Tatoeba, SCOWL, Leipzig Corpora, OpenTaal, …) are listed in THIRD-PARTY.md.
  • Fixed: preview ↔ projector wrap matching — lines now wrap at the identical points on the operator preview and the fullscreen projector, so the line you cue is the line the talent reads. The projector renders the script on a fixed-size stage that is CSS-scaled to the monitor, so a line break can no longer land in a different place on the big screen.
  • Fixed: seek-bar elapsed timer — the elapsed-time readout no longer climbs past the total read time; it clamps at the end instead of counting up forever once the script has fully scrolled.

0.900.0 — Studio management & workflow (Phase 8)

Own your studio. Phase 8 makes a whole layout portable and shareable, adds version control for your scenes, puts your most-used actions one tap away, and opens up full theming and a USB-stick portable mode. Everything is strictly local — no new dependencies and no new network calls; the pack, backup, snapshot, and theme features are plain local files, and the quick-actions LAN sharing rides the existing opt-in web panel.

  • Scene-collection pack — export/import — bundle a whole layout (the scene graph plus its local assets: images, media, LUTs, masks, fonts) into a single portable .fcappack and share it as one file. Import relinks assets on the target machine; URL/UNC paths are recorded as external and never statted, and fonts are flagged for attribution. No marketplace, no server.
  • Scene-collection diff & merge — compare the active collection against another collection, a .fcappack, or a snapshot; see added / removed / changed scenes, sources, and per-item transform / filters / visibility; then cherry-pick exactly which changes to merge in (a picked scene brings the sources it uses). Reconcile two versions without a cloud.
  • Named version snapshots — one-click named checkpoints of the active collection with a browsable timeline, restore, and a diff view (reusing the compare tool). Intentional history beside autosave and undo; capped at 50 with disclosed disk usage, and a "before restore" checkpoint so a restore is never one-way.
  • Portable mode — drop a freally-portable.txt marker next to the executable and all app data (settings, profiles, collections) lives under the exe's folder — run the whole studio from a USB stick without touching the host OS profile. Settings → About shows the portable status and the data location.
  • One-file backup & restore — export everything (settings, profiles, scene collections) to a single versioned .fcapbackup, and restore it selectively here or on a new machine (a migration flow). Secrets — stream keys, passwords — are always stripped from a backup; server-owned fields (EULA, onboarding, camera profiles) are preserved on restore.
  • Theme editor — a visual editor over the CSS-variable theme: author the full six-colour palette with live WCAG contrast warnings, and export/import shareable .fctheme files.
  • Dock layout presets — save named view layouts (stats-dock visibility + mixer orientation) and switch between them.
  • Settings search & recently-changed — a search field over the Settings modal (every category keyword-indexed, jump-to) plus a per-category "changed since opened" marker.
  • Quick-actions grid — a customizable Stream-Deck-style button board (scenes, commands, macros, soundboard pads) with per-button colour and pages; command buttons fire the same allowlisted studio actions as hotkeys and the LAN panel, and the grid config is shared with the CAP-N06 LAN panel.
  • Fixed: teleprompter projector — "Open projector" opened a blank, borderless window that could not be closed (its IPC was unauthorized). It now opens as a normal, closable window that shows the scrolling script, and Esc closes it; File → Exit quits reliably too (the main window was missing the window-close permission).
  • Fixed: teleprompter dialog — a Stop button (pause + rewind to the top) so a short script no longer looks blank after Play and the text can be re-read and re-edited.
  • Fixed: sources panel — the per-source remove and reorder controls were hover-only and could be clipped off the rail; each source now has an always-visible top control strip (LIVE/REC · lock · reorder · filters · properties · clone · remove).

0.800.0 — Remote guests & show production (Phase 7)

From "guests connect" to "a produced show." Phase 7 turns the remote-guest session into a multi-guest studio and adds the live-production layer on top: draw on the live program, run a teleprompter, hold guests in a green room before they go on air, send them private cues, watch each guest's connection quality, and reflow everyone into an automatic grid. The remote-guest transport now admits up to eight guests (host + 8 = a 1–9 gallery). Everything rides the existing opt-in P2P session — no new brokers, no server Mike runs, nothing that auto-connects — and every new surface is off or unbound by default.

  • Telestrator — draw on the live program — annotate the outgoing feed with pen, highlighter, arrow, and ellipse tools, composited into the program so the marks land in the recording and the stream at the moment you draw them, not just your preview. Marks fade after a set time, or switch to whiteboard mode to keep them until you clear. Tablet-pressure aware, with a clear hotkey (unbound until you assign it); drawn straight over the finished program, so it reaches every output — including the stream-only / recording-only variants.
  • Teleprompter — a smooth-scrolling script on three surfaces at once: an in-app panel, a fullscreen projector for the talent (with mirror mode for beam-splitter glass), and a view on the LAN touch panel. Plain-text or light-Markdown scripts with adjustable speed and font; scroll control from a hotkey, MIDI, OSC, or the LAN panel — the scroll offset is shared, so every surface stays in step.
  • Green room for remote guests — new guests can land in a private holding state: you see their camera and a cam/mic tech-check in a monitor, off-program, and seat them on air in one click. Optional and off by default; the guest sees a "you're in the green room" banner on both the desktop app and the web join page.
  • Cue & talk system — send a connected guest private text cues ("30 seconds", "wrap up", "you're next", "speak up") over the existing P2P data channel. The guest sees the cue with a live countdown on their join page or app — never in the program. No new network surface.
  • Guest connection-quality dashboard — a live per-guest QoS roll-up: round-trip time, packet loss, received resolution and framerate, and jitter, with a green/amber/red badge and a history sparkline, so "your connection is degrading" is visible before viewers notice.
  • Auto-grid guest layouts — arrange the seated guests into an automatic 1–9 grid that reflows as guests join, leave, or are seated, with each seat's name published as a {{guestN}} variable so a title source becomes an auto-updating nameplate. Non-overlapping by construction.
  • Multi-guest remote sessions — the opt-in remote-guest transport now admits up to eight guests at once (previously one), each its own tile with its own mute gates, connection quality, cue channel, and one-click seating. Still the same peer-to-peer session — media flows directly between host and guests, no server, and nothing connects until you invite someone.

0.700.0 — Streaming reliability & insight (Phase 6)

Rehearse, measure, explain. Phase 6 is about knowing your show will hold up before you go live, and understanding what happened after: run the whole pipeline against a private local sink with a simulated bad network, benchmark what your machine can actually sustain, watch a recorded timeline of every render/encode/reconnect event, and get a written report when the stream ends — plus flag any source as stream-only or recording-only. Everything is off or neutral by default, and nothing here adds a network call — the dry run and benchmark are fully local, and the network simulator only ever degrades the rehearsal's own loopback sinks, never a real broadcast.

  • Network condition simulator — a rehearsal-only shaper: deterministic bandwidth caps, added latency/jitter, and scheduled outages applied to the dry-run sinks, so you can drill reconnects, multistream lane isolation, and bitrate choices against a thin, flapping uplink. "Hotel Wi-Fi" and mobile-hotspot presets plus a fully custom profile; by construction it can never touch a real Go Live.
  • Dry-run mode / "Go Live (rehearsal)" — run the whole live pipeline (compositor, encoder, mux, and the real reconnect logic) against an owned loopback sink with the real stats dock, LIVE-state UI, and alarms, with zero bytes leaving the machine. A distinct violet Rehearse button, a REHEARSAL badge on every live surface, and a state banner make it impossible to confuse with a real broadcast; stream keys are never even read.
  • Dropped-frame forensic timeline — a session-long recorded timeline correlating render time, encoder queue depth, per-target bitrate and drops, reconnects, scene switches, markers, and alarms on one zoomable graph. The stats dock shows now; this explains the whole session, sampled off the render loop so the 60 fps compose path gains no work.
  • Stream session report — on session end, an optional local HTML + Markdown report (uptime, average and peak bitrate per target, down-time percentage, reconnect log, markers, alarms, recording paths) written next to the recording. Local only, shareable by you alone; off by default.
  • Encoder benchmark wizard — a guided, fully-offline benchmark that actually runs short encode ladders (every detected encoder × preset × resolution × fps) on your machine, measures achieved fps and headroom, and recommends canvas/encoder/bitrate settings — with one-click apply. Failures are listed, never hidden. Refused while live or recording so it never steals the encoder mid-show.
  • Per-output source visibility — flag any scene item as visible on the stream but not in the recording, or vice-versa, within a single canvas — keep alerts and chat on the live feed while the VOD stays clean. Hover LIVE / REC toggles on each source row; your preview, projectors, and multiview always show the full program.
  • Looping text & advertising effects — three item filters for unattended promo loops: a Perspective filter that tilts any source (text, cameras, captures) away as a flat plane with an optional far-edge fade — chain it after the endless Scroll filter for a full "Star Wars" opening-crawl — and a Fade Loop filter that fades an item in, holds, fades out, and holds hidden on a repeating cycle. All time-driven, so an ad runs itself.

0.600.0 — Recording & replay depth (Phase 5)

Everything around the file. Phase 5 builds out what happens to a recording before, during, and after it lands on disk: isolated per-source files, a clip trimmer, real transparency, event-driven splitting and chapter markers, an after-recording task pipeline, an integrity verifier, and SMPTE LTC timecode. Every new capability is off or neutral by default — an untouched recording is bit-identical to before — and nothing here adds a network call.

  • Per-source ISO recording — record selected sources clean to their own files alongside the program, pre-composite, at canvas size and frame rate so every ISO drops onto an editing timeline aligned with the program. Each remote guest is just a source, so per-guest ISOs come free.
  • Replay & clip trimmer — a trim window for any saved recording with frame-step in/out points and a live preview; an honest badge says up front whether the clip will stream-copy (a keyframe-aligned cut — instant, lossless) or re-encode. One click reframes to a 9:16 vertical master.
  • Alpha-channel recording — record the program with real transparency to the owned lossless .frec (the preview and stream keep the normal opaque program), and export a ProRes 4444 or QuickTime Animation .mov master for editing.
  • Split on scene change / marker — start a new file on a scene switch, a chapter-marker drop, or a rundown step, each part beginning exactly on the event's frame boundary.
  • Auto-markers from events — chapter markers dropped automatically by studio events (scene switches, replay saves, reconnects, dropped-frame bursts, alarms, rule actions) with typed labels, alongside the manual marker hotkey; written as mkv chapters or a sidecar.
  • Post-record pipeline — a per-profile after-recording task chain from a closed, sandboxed action set (verify → remux → normalize → rename → move/copy → reveal → notify scripts) that runs in the background with a live queue and stops at the first failure. No arbitrary commands, by design.
  • Recording integrity verifier — a fast local pass that checks container structure, frame and per-track continuity, A/V interleave, and duration versus wall-clock, with a plain-language OK / warning / fail report — catch a broken file minutes after the show, not during the edit.
  • SMPTE LTC timecode source & reader — an owned linear-timecode generator on an assignable track and a reader on any audio input, driving a burn-in timecode line and LTC-stamped markers. Classic audio timecode, fully offline, accurate to ±1 frame.

0.500.0 — Audio production depth (Phase 4)

From mixer to console. Phase 4 turns the shipped mixer into a broadcast console: physical output routing and aux/cue mixes, an N×M ducking matrix, a gain-sharing auto-mixer, EBU R128 loudness normalization (live rider + post-record pass), a parametric EQ with a live spectrum, a de-esser and rumble guard, a soundboard, audio-only (podcast) recording, and per-guest mix-minus with one-click voice-chain presets — plus the MIT-clean foundation for CLAP and VST3 plugin discovery. Every audio filter becomes a live visual instrument. Owned classic DSP (no ML); everything ships off/neutral by default, so an untouched mix is bit-identical to before.

  • Aux buses & physical output routing — route the master or any of the six track buses to extra output devices, each with a trim, from a Routing matrix. A track bus sent to headphones is a cue mix that differs from the monitor.
  • Sidechain ducking matrix — a grid of who ducks whom, each pair with its own depth, threshold, attack, and release; a strip can be ducked by several triggers at once.
  • Gain-sharing auto-mixer — a Dugan-style per-strip toggle that holds the total gain of the auto-mixed mics constant and hands it to whoever is speaking, for multi-host panels and podcasts.
  • EBU R128 loudness normalization — a live rider steers the program toward −14 / −16 / −23 LUFS with a peak ceiling, plus a post-record Normalize action on the recordings list.
  • Parametric EQ + spectrum view — an N-band parametric EQ (bell / shelf / notch / HP / LP) with draggable nodes over a live spectrum analyzer, the same RBJ math the engine runs.
  • De-esser + rumble guard — a split-band de-esser (a sibilance detector drives a dynamic high-shelf cut, no ML) and a clean low-cut rumble guard for desk thumps and plosives.
  • Soundboard — a pad grid of local clips with per-pad gain, choke groups, loop, hotkey, track assignment, and optional auto-duck of the rest of the mix.
  • Audio-only recording — a podcast mode that records per-track WAV via the owned writer (no video encoder), optionally transcoded to FLAC/Opus, with silent-skip markers.
  • Per-guest mix-minus & voice-chain presets — one-click gate→comp→EQ→limiter chains and an owned N−1 return per source so a remote guest hears no echo.
  • Live filter visualizers — every audio filter shows a transfer or frequency-response curve with in / out / gain-reduction meters that move with the audio.
  • Audio plugins — CLAP + VST3 discovery — a Plugins panel finds your installed CLAP and VST3 plugins locally (both now MIT-licensed); crash-isolated hosting is in progress and stated plainly.
  • Desktop shortcut — the Windows MSI now creates a Freally Capture desktop shortcut (removed on uninstall).

0.300.0 — New Sources & Overlays (CAP-N Phase 2)

Ten new things to put on the canvas — all local. Instant replay rolls the buffer into the show, a gapless media playlist replaces "one file at a time", titles and scoreboards get a real layered designer with live control, and two LAN sources land: an SRT/RTMP listener and Freally Link, an owned instance-to-instance share that needs no ffmpeg on either end. The network invariant holds: both LAN features are off until you add them, bind only the local machine, and never touch the internet — and the input overlay and cursor effects read input only while enabled and never log or store a keystroke.

  • Instant-replay playback source (CAP-N10) — a hotkey snapshots the armed replay buffer (stream copy, no re-encode) and plays it into the program at 100 / 50 / 25 % — retimed, never frame-interpolated — with scrub and pause; at the end the source clears back to transparent so the show returns to live by itself. Slow motion is silent by design: retimed audio would smear, and we don't fake it.
  • LAN ingest source (CAP-N11) — a built-in SRT/RTMP listener: point any phone camera app, encoder, or second PC at the URL (shown with a QR code) and it lands in a scene with its own mixer strip. SRT carries an optional AES passphrase; RTMP has no authentication of its own — the form says so and prefers SRT.
  • Freally Link (CAP-N12) — two Freally instances on one LAN: enable the Link output on one machine with a pairing key, hit "Scan the LAN" on the other, type the key, and the program appears as a source. A receiver without the key never sees a frame. An owned wire protocol + mDNS discovery — no ffmpeg, no broker, no server.
  • Input overlay source (CAP-N13) — live keyboard, mouse, and gamepad visualization with four layouts (WASD, compact keyboard, dual-stick pad, 8-way fight stick). Reads input only while the source is enabled; never logs or stores anything. Keyboard/mouse are Windows-first — said honestly in-product; gamepads work on all three OSes.
  • System-stats overlay source (CAP-N14) — the stats dock's real measured numbers (fps, CPU, memory, GPU compose time, dropped frames, live bitrate) composited for your viewers, with per-line toggles. GPU utilization is deliberately absent: we don't measure it, so we don't guess.
  • Audio visualizer source (CAP-N15) — FFT spectrum bars, an oscilloscope, or stereo VU meters bound to any mixer strip, track bus, or the master mix — tapped post-fader, so a muted source visualizes flat, exactly as it sounds. Owned radix-2 FFT; no ML anywhere.
  • Title & scoreboard designer (CAP-N16) — layered templates (text, image, shape) with outline and drop-shadow text, animate-in/out (fade, slide, wipe), fields bound to a watched file (CSV cell, JSON pointer) or a studio variable, and live control — swap a name, edit the score, fire the animation — without touching the scene.
  • Media playlist source (CAP-N17) — an ordered playlist that plays truly gaplessly (the whole trimmed list runs through one decode, so item boundaries are frame-exact), with per-item in/out trims and cue points, loop / shuffle / hold-last, next/previous hotkeys, and a "now playing" variable any Text source can show.
  • Speedrun split timer source (CAP-N18) — import a LiveSplit .lss file and run it: personal-best / best-segments / average comparisons, gold-segment highlights, and global split / undo / skip / reset hotkeys. The file is read only — a run is never written back, and process-memory auto-splitters are deliberately not supported.
  • Cursor highlight & click effects (CAP-N19) — a cursor halo, colour-coded left/right click ripples, and optional keystroke ghosting drawn onto display and window captures, on the owned cursor path. Windows-only, honestly; keystrokes are drawn, never logged.

  • Display Capture, Window Capture, and Video Capture Device pickers in the Sources rail — the selected source previews live with a resolution / fps / dropped-frames overlay.
  • Per-OS capture behind one owned interface, with an honest dropped-frame count and permission flows that never pretend (macOS deep-links to the right Privacy pane; the Wayland picker explains portal-only capture).
  • Windows mouse-pointer blending in display capture; window capture survives resizes and display-mode switches.
  • Everything stays local — no accounts, no telemetry, zero outbound network calls.

This is an early development build: capture and the live preview work; the GPU compositor, audio mixer, recording, and streaming arrive in the milestones below.

0.10.0 — Foundation & repo (done)

The cross-platform studio shell is built: a Tauri v2 + React (Havoc dark) UI over the owned Rust workspace (capture/compositor/scene/audio/encode/stream crates), the typed UI↔core bridge with a live stats feed, a persisted settings store, the 3-OS CI matrix, per-OS packaging (the Windows build is a GUI app with no console window), and this site.

Planned for the first public build (0.70.0 — Studio MVP)

  • Build scenes from sources on the owned wgpu GPU compositor, with per-source filters and transitions.
  • Multi-track recording via hardware encoders (NVENC/Quick Sync/AMF/VAAPI/VideoToolbox) + x264 fallback, plus the owned lossless freally-video codec.
  • Single-target live streaming (RTMP/RTMPS) to Twitch/YouTube/Kick/Facebook/Trovo/custom, with auto-reconnect and stream delay.
  • Studio Mode (preview/program) + transitions, a virtual camera, global hotkeys, profiles + scene collections, and a live stats dock.
  • No account, no telemetry, no cloud — streams go direct to the platform.

The detailed build plan is maintained privately; the public ladder lives in the roadmap below.
