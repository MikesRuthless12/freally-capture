0.96.0 — Accessibility, 18 languages, onboarding & themes

The launch polish. The studio now speaks eighteen languages, is fully operable from the keyboard and audible to a screen reader, configures itself on first run, and can be themed light. Two of the bugs fixed below would have shipped in 1.0.0, and both were invisible in review — each is now caught by a lint or a test rather than by memory.

  • The studio speaks 18 languages — ar de en es fr hi id it ja ko nl pl pt-BR ru tr uk vi zh-CN, from a catalog of 989 strings. A fresh install follows your operating system’s language; an explicit choice wins. Arabic drives a right-to-left layout. English is layered beneath every locale, so a string a translator has not reached yet reads in English rather than as a raw identifier.
  • Operable without a mouse, audible without a monitor — every keyboard stop draws a visible focus ring, dialogs trap Tab and hand focus back where they took it, and going live, recording, reconnecting, and losing a second of video to dropped frames are announced to screen readers. The system “reduce motion” setting is honoured.
  • A first-run wizard that configures the machine it is actually running on — it enumerates your GPUs and physical cores, ranks the available hardware encoders, and proposes an encoder, canvas, frame rate, and bitrate with the reason stated in your own language. An encoder your driver has already refused is skipped rather than recommended. Skipping counts as finishing: the wizard never returns uninvited.
  • A command palette on Ctrl/Cmd+K, matching on subsequences, so ssc finds “Start Screen Capture”.
  • One Settings modal and an About page — language, theme, and the stats dock in one place; version, build metadata, licences, and the bug reporter in the other.
  • Light and custom themes. The accent colour is validated as a plain hex triple on both sides of the boundary, because a string written into a stylesheet is an injection surface.
  • .frec recordings carry their own icon in Windows Explorer, rather than the studio’s.
  • Fixed: upgrading re-ran the first-run wizard. The new “onboarding complete” flag defaults to false, so every existing install would have been greeted by a wizard whose starter template adds a second display capture on top of the scene you had already arranged. Settings files are now migrated on load: one that predates the flag but has already accepted the licence has plainly been run before.
  • Fixed: the light theme rendered parts of the studio white-on-white. Panels are drawn with translucent white over near-black, and a colour variable cannot reach inside a utility class — so the light theme re-tints those classes with translucent black. Six were missed, including download progress-bar tracks and the mixer’s mute and gate chips. A lint now fails the build on any that has no light override.
  • Fixed: choosing an accent colour threw a light-theme user into the dark.
  • Fixed: screen readers announced a frame-drop burst on every reconnect — the dropped-frame count is cumulative and kept climbing through the outage, so the first moment back on air reported the whole outage as new.
  • Fixed: a failed settings save could discard an unrelated setting that had succeeded in the meantime.

0.95.1 — Crash reporting, EULA persistence & dialog fixes

Six bugs found while drilling the bug-report flow on real hardware — two of which would have shipped in 1.0.0 — plus the crash-reporting and updater work those drills demanded. A crash now tells you it happened and brings the studio back with the report already open.

  • A crash tells you, and brings the studio back — a dying app cannot show its own error window, so the panic hook spawns the same executable as a tiny helper, which shows a native “stopped unexpectedly” dialog, waits for the crashed process to exit, and relaunches. The reopened studio surfaces the scrubbed report automatically.
  • Accepting the EULA no longer un-accepts itself — changing any setting in the same session you accepted wrote a stale null back over your acceptance, and the gate returned on the next launch. Acceptance is now write-protected at the store.
  • Dialogs no longer render off-screen — a backdrop-blur on the dock panels made them the containing block for position: fixed, so 16 of 19 dialogs centred inside their dock instead of the window, with their buttons out of reach. Check for updates was one of them.
  • Emailed crash reports work — URL length was counted in characters, but percent-encoding inflates a 3-byte character ninefold, so a real backtrace produced a URL past Windows' limit, where it opens a blank window and reports nothing. The whole URL is bounded now.
  • Compose in Gmail — beside the mail-client button. Plain https, no API key, nothing auto-sent.
  • Update prompt at launch — the studio checks once per launch and shows the new version with its real release notes in a read-only field, with an explicit yes/no. A pending crash report takes precedence; an offline check stays silent.
  • Dropdown menus close when you click away — the Sources + menu and both Add filter menus. Escape closes the innermost surface only.
  • Windows installer — updates now run the NSIS wizard with a Run Freally Capture checkbox on the Finish page, and the installer carries the app icon instead of NSIS's stock one.

The Force a test crash button is gone: a control that crashes the app has no place in a live studio. Also fixed: a shell-injection sink in the release workflow, where a tag name was spliced into a shell script before bash parsed it.

0.95.0 — Game capture, distribution & per-app audio (Phase 8)

Signed self-hosted auto-update, per-application audio capture, an NDI runtime seam, a game-capture seam that flags the anti-cheat risk honestly, and a first-run EULA gate — the “toward 1.0.0” surface, all local and off by default. This is also the release that publishes the extensibility surface built in 0.90.0.

  • Signed self-hosted auto-updater — the app checks a signed latest.json on the GitHub releases endpoint and verifies every download against a minisign public key baked into the binary before applying it; an unsigned or tampered package is refused. ⭳ Check for updates… lives in the Controls dock — nothing downloads without a click.
  • Per-application audio capture — capture one app's audio as its own mixer source, Windows-first via WASAPI process loopback (Win10 2004+). It gets a full mixer strip: VU, fader, mute, monitoring, filters, track. The COM unsafe is isolated in its own crate so the audio engine stays #![forbid(unsafe_code)]. Linux/macOS show honest per-OS guidance.
  • Game Capture seam, flagged honestly — injecting a DX/GL/Vulkan hook is an opt-in, flagged milestone: the app never injects silently and surfaces the anti-cheat ban risk before any hook. The working path today — Window Capture (borderless/windowed) — is recommended per-OS.
  • NDI runtime seam — optional NDI output detects a user-installed NDI runtime (never bundled) and link-probes it; the output flag is gated on that detection.
  • First-run EULA gate — the studio doesn't render until the current EULA version is accepted; EULA.md is embedded at build time, so the accepted text is exactly what ships.
  • Video in the stream — embed a video into the scene and pause/resume it live while talking over it; the media source's audio pauses and resumes in the mix with the picture.
  • In-app component downloads — the on-demand ffmpeg download shows an explicit percentage beside its bar. A new Browser Source runtime (Chromium/CEF) component downloads the ~100 MB runtime the same honest way: it resolves the newest stable build from the official CEF index and verifies the download against that index's SHA-1 before unpacking — no bundling, no hardcoded hash.

Scoped honestly: VST2/3 hosting is deferred behind a flag — the VST2 SDK is no longer licensed by Steinberg and VST3 is GPLv3-or-proprietary, both conflicting with a $0/no-extra-license build; the owned DSP filter set is the shipped alternative. Game-capture GPU hooking, NDI frame-send, and the browser source that renders through the downloaded runtime are the named follow-on milestones these seams are stable for.

0.90.0 — Remote control, scripting & plugins (the extensibility surface)

Drive the studio from a Stream Deck, react to going live with a script, embed a chat dock, and extend the app with a plugin — all local and off by default.

  • WebSocket remote-control API — a password-protected socket a Stream Deck / Companion-style controller connects to: switch scenes, run the transition, start/stop the stream and recording, save replays, set mutes/volumes. Off by default, this-machine-only unless you opt into LAN, and the password never crosses the wire (challenge–response). It can't read files.
  • Browser docks — open a chat popout, alerts page, or Companion web buttons as their own window beside the studio. The docked page renders and gets no access to the app.
  • Sandboxed Lua scripting — .lua scripts that react to go-live / scene / recording events and drive the same commands as the remote API. No file or OS access; a script error can't touch your stream or recording.
  • Plugin SDK — documented Source/Filter contracts + a sample plugin, so the community can add sources and filters without touching core.
  • Plus the web join page for phone guests (scan the invite QR, join from a browser — no install) and a Windows native-preview + cursor-capture polish pass.

The true browser source (a full Chromium engine) and the virtual camera (a signed OS driver) each follow as their own on-demand-component milestones.

0.70.0 Jul 6, 2026 · Streaming + Studio MVP
Releases

0.70.0 — Streaming + Studio MVP (first public)

The one that makes it a studio you can go live with. Single-target streaming over RTMP/RTMPS to Twitch, YouTube, Kick, Facebook, and Trovo with auto-reconnect; Studio Mode (preview/program) with GPU transitions; global hotkeys; switchable profiles + scene collections; a real fps/CPU/GPU stats dock — and opt-in P2P remote guests. The local recording keeps running no matter what the stream does.

Live streaming

  • Go Live single-target over RTMP/RTMPS to Twitch / YouTube / Kick / Facebook / Trovo / custom ingest — direct to the platform, no restream server, no account. Auto-reconnect with backoff rides out a hiccup.
  • The stream key is treated as a secret — masked in the UI, never logged, and scrubbed out of any error message.
  • A pulsing LIVE button with an HH:MM:SS elapsed clock and honest reconnecting/failed states; an optional auto-record on Go Live. The local recording is sacred — a stream dying can never touch it.

Studio Mode + control

  • Studio Mode: edit a preview scene, then commit it to the program through a GPU transition — cut, fade, slide, swipe, or a luma wipe — so the audience sees only the finished blend.
  • Global hotkeys for record, Go Live, and transition; switchable profiles (settings) + scene collections; and a stats dock now showing real fps, dropped frames, GPU render time, and CPU/memory.
  • Remote guests (opt-in P2P): host up to three guests over WebRTC — their camera + mic land in your scene, positionable into corners with one click, with a two-gate mute and screen-share hand-off. Free by default; media stays peer-to-peer.

The virtual camera follows as its own milestone right after 0.70.0 — a real virtual camera is a signed OS driver component, built and signed on its own. Multistream to several platforms, SRT/WHIP, and the replay buffer arrive in 0.85.0.

0.56.0 Jul 6, 2026 · Native GPU preview + Window Capture
Download v0.56.0

0.56.0 — Native GPU preview ("OBS feel") on all three OSes (done)

The preview now feels like OBS: the compositor's GPU output is painted straight onto the screen — no read-back, no encode round-trip, no lag — on Windows, macOS, and Linux. Window Capture grew up alongside it: a live-thumbnail picker, minimized windows listed, sources that re-bind to the same window after a restart, and captures that recover on their own.

Native GPU preview

  • A zero-copy native GPU preview surface replaces the read-back → JPEG → canvas preview with the compositor's own GPU output painted directly on screen — per OS: Windows DirectComposition (DX12, verified real-time on hardware), macOS CAMetalLayer (Metal), Linux an X11 child window on Vulkan/GL (Wayland keeps the JPEG preview for now).
  • The interactive selection box + transform handles are drawn into the GPU frame itself — preview-only, never recorded or streamed.
  • All three OSes are CI-proven to render: the screenshot smoke asserts the actual pixels on every push. Under the hood the GPU stack was upgraded wgpu 0.20 → 27.
  • The JPEG preview stays as the universal fallback and returns automatically whenever the native surface isn't viable (a non-DX12 Windows GPU, Wayland, a lost surface).

Window Capture

  • The picker shows a live thumbnail of every open window — refreshed while it's open, delivered as in-memory images over IPC, never written to disk.
  • Minimized windows are listed too (previously filtered out) — on all three OSes, backed by a dedicated window-enumeration CI job.
  • A saved Window source re-binds to the same window when you relaunch — it stores the window's durable identity (app, class, title) and re-resolves the live window on start, the way OBS does; old handle-only scenes still load.
  • Errored captures auto-recover (display / window / camera, with per-source backoff) the moment the window reopens or the device reconnects — no manual Retry needed.
  • The yellow Windows capture border is gone (best-effort on older Windows builds).

Next: 0.70.0 (studio MVP) — single-target streaming, Studio Mode + transitions, hotkeys, profiles, and remote guests (above).

0.55.0 Jul 3, 2026 · Audio mixer + recording
Download v0.55.0

0.55.0 — Audio mixer + recording (done)

The studio can now record. Its program feed writes to disk multi-track, with the best available hardware encoder or the owned lossless codec — and the audio engine (up to 6 tracks, owned classic-DSP filters, LUFS) that shipped alongside feeds those tracks. Everything runs on the machine — no AI/ML anywhere, no accounts, no cloud.

Recording

  • The owned freally-video (.frec) lossless codec — the default local format, authored here and owned outright (temporal deltas + prediction + an owned LZ77 stage), nothing fetched, up to 6 PCM audio tracks, and a truncation-tolerant reader (a crashed recording still plays back).
  • Per-OS hardware encoders (NVENC / Quick Sync / AMF / VAAPI / VideoToolbox) with a universal x264 / x265 / AV1 CPU fallback — detected, best-picked, and confirmed by a real smoke encode on first use.
  • Multi-track mp4 / mkv / mov / webm (or lossless .frec), automatic file splitting, and gapless pause / resume — with a REC indicator showing duration + track count.
  • Encoder settings + presets (CBR/VBR/CQP, Quality/Balanced/Performance), HEVC/AV1 recording, and post-record remux to MP4 (no re-encode).
  • A Media source — play a video/image file on the canvas with its audio in the mixer (.frec through the owned codec; other formats through the labeled ffmpeg component).
  • The patent-encumbered wire codecs run through a clearly-labeled, on-demand ffmpeg component — never bundled: fetched only when you ask, SHA-256-verified before it ever runs. The owned lossless path needs none of it.

Audio mixer

  • Audio Input (mic / line-in) and Audio Output (desktop audio) capture — honest per OS: Windows WASAPI loopback, Linux monitor devices, macOS a virtual device (never silently installed).
  • A mixer with per-source volume/mute/monitor across up to 6 tracks, owned classic-DSP filters (spectral denoise — no ML, gate, compressor, limiter, 3-band EQ, gain, ducking), push-to-talk/mute hotkeys, and a broadcast-standard LUFS meter.

Next: 0.70.0 (studio MVP) — single-target streaming, Studio Mode + transitions, and the virtual camera. The native GPU preview (real-time "OBS feel") landed as its own milestone in 0.56.0, above.

0.40.0 Jul 2, 2026 · Compositor + scenes/sources
Download v0.40.0

0.40.0 — Compositor + scenes/sources (done)

The heart of the studio is in. Every scene composes on the owned wgpu GPU compositor — per-item move/scale/rotate/crop, seven blend modes, sustained 60 fps at 1080p — driven by the owned scene/source model: scenes share sources, items stack by z-order, and the whole collection autosaves locally and restores on launch. Each item carries an ordered on-GPU filter chain: chroma key (with spill suppression), color correction, LUT (.cube), blur, image mask, sharpen, scroll, and crop, all live-editable.

Highlights

  • Scenes and Sources rails work end to end: create/rename/reorder scenes; add displays, windows, webcams, images, colors, and text; toggle visibility, lock, and stack order; share one source across scenes.
  • Pixel-accurate on-canvas editing — drag, scale from corners or edges, rotate (with 15° snapping), and Alt-drag any edge to crop, straight on the program preview.
  • The Text source ships the complete Noto Sans family in the app (upright + italic + Arabic + Hebrew, every weight) with real shaping and RTL — text renders identically on every machine.
  • Honest per-source status (starting / live / error) and honest limits: a machine with no usable GPU says so instead of freezing.
  • Everything stays local — scene collections are plain JSON on your disk; frames never touch a socket.

This is an early development build: scenes, capture, and the compositor work. The audio mixer has since landed on main (above); recording and streaming arrive in the milestones below.

0.25.0 Jul 1, 2026 · Capture core
Download v0.25.0

0.25.0 — Capture core (done)

Real capture is in. Screens, windows, and webcams stream live into the program preview on all three OSes: Windows via DXGI Desktop Duplication (displays, with the mouse pointer blended in) and Windows.Graphics.Capture (windows); macOS via ScreenCaptureKit (12.3+), with the Screen-Recording permission requested honestly and a Settings deep-link on denial; Linux via the ScreenCast portal on Wayland — the system dialog picks the screen or window, the only capture Wayland allows — plus a direct X11 path. Webcams and capture cards enumerate with per-device formats and stream live. The preview pipe is in-process end to end: frames never touch a socket or disk.

Highlights

  • Display Capture, Window Capture, and Video Capture Device pickers in the Sources rail — the selected source previews live with a resolution / fps / dropped-frames overlay.
  • Per-OS capture behind one owned interface, with an honest dropped-frame count and permission flows that never pretend (macOS deep-links to the right Privacy pane; the Wayland picker explains portal-only capture).
  • Windows mouse-pointer blending in display capture; window capture survives resizes and display-mode switches.
  • Everything stays local — no accounts, no telemetry, zero outbound network calls.

This is an early development build: capture and the live preview work; the GPU compositor, audio mixer, recording, and streaming arrive in the milestones below.
