Getting started

Freally Capture is in development. As of 0.700.0, you can build scenes on the GPU compositor, mix audio across up to 6 tracks (including per-application audio), record multi-track to disk, go live (single-target and simultaneous multistream over RTMP/RTMPS/SRT/WHIP) with Studio Mode, a replay buffer, vertical/multi-canvas, nested scenes, and a no-key chat overlay — extend it all with the remote-control API, browser docks, Lua scripting, and a plugin SDK — and keep it current with a signed self-hosted auto-updater. The preview is a native GPU surface ("OBS feel") on Windows, macOS, and Linux. The interface speaks 18 languages, is fully keyboard-operable and screen-reader-audible, sets itself up with a first-run wizard, and can be themed light, dark, or custom. The first CAP-M must-have batch (0.97.0) adds scene-editing depth — multi-step undo/redo, an OBS scene-collection importer, a missing-file doctor, a multi-select with align/distribute and custom guides, copy-paste transforms, a keying workbench — and monitoring — a multiview and fullscreen projectors for any scene/source/program/preview on any display, plus a one-key still grab. The second batch (0.98.0) adds broadcast safety & reliability: a go-live pre-flight checklist, always-on safety alarms, a source health dashboard, mid-session encoder failover, crash-safe recording with a next-launch repair, a quit guard with an ordered shutdown, a panic button, recording filename templates, and a redacted diagnostics bundle — see Broadcast safety. The third and final batch (0.99.0) closes the must-have set with sources, devices & calibration: built-in test signals (SMPTE bars, calibration grid, motion sweep, lineup tone, A/V flash+beep), a guided A/V sync calibration workbench, timer & clock sources, text bound to a watched file (.txt/CSV/JSON), a searchable hotkey map with conflict detection, mixer pan/solo/mono, per-device deinterlacing, and cross-platform camera controls with per-device profiles — see Sources & devices. The automation + capture & device-depth release (0.200.0) makes the studio drive itself and finishes the glass-to-canvas path: an automation rules engine, macros with variables & sequences, MIDI and OSC control surfaces, hotkey chords & layers, a LAN touch panel and a tally-light service, PTZ camera control, and a show rundown — plus a low-latency passthrough monitor, pixel-perfect scaling, punch-in zoom & follow pan, auto black-bar crop, window↔app-audio auto-linking, and HDR→SDR tone-mapping, alongside a scene backdrop and true reverse media playback. Every automation surface routes through one fixed, off-by-default command allowlist, so none can name a file or reach the internet. The new-sources & overlays release (0.300.0) puts ten new things on the canvas: an instant-replay playback source (roll the armed buffer into the program at 100/50/25 % — retimed, never interpolated), a truly gapless media playlist with per-item trims and cue points, a layered title & scoreboard designer with live control, an audio visualizer (FFT bars, oscilloscope, stereo VU) tapped post-fader, a system-stats overlay of real measured numbers, a LiveSplit-compatible speedrun split timer, a keyboard/mouse/gamepad input overlay, cursor highlight & click effects, an SRT/RTMP LAN ingest listener with a QR-coded URL, and Freally Link — instance-to-instance program sharing over your own network, gated by a pairing key, with no ffmpeg on either end. Both LAN sources are off until added, bind only the local machine, and never touch the internet; the input overlay and cursor effects read input only while enabled and never log a keystroke. The compositor & FX depth release (0.310.0 → 0.400.0) adds 3D corner-pin, downstream keyer layers, freeze-frame, source cloning, an expanded blur family, a move (morph) transition, per-scene-pair transition rules, a WGSL shader studio, a bezier mask editor, and track-matte stingers with an optional program-audio duck. The audio production depth release (0.500.0) turns the mixer into a broadcast console: physical output routing and aux/cue mixes, a ducking matrix, a gain-sharing auto-mixer, EBU R128 loudness normalization, a parametric EQ over a live spectrum, a de-esser and rumble guard, a soundboard, audio-only (podcast) recording, per-guest mix-minus with voice-chain presets, and CLAP + VST3 plugin discovery — every audio filter now a live visual instrument with in/out/gain-reduction meters, all owned classic DSP, off by default. The recording & replay depth release (0.600.0) builds out what happens to the file: per-source ISO recording (clean per-source and per-guest files, timecode-aligned to the program), a replay & clip trimmer with a stream-copy-vs-re-encode badge and a one-click 9:16 reframe, alpha-channel recording to the owned lossless format with ProRes 4444 / QTRLE master export, event-driven split on scene change / marker / rundown step, typed auto-markers from studio events, a closed sandboxed post-record pipeline (verify → remux → normalize → rename → move/copy → reveal → notify scripts), a recording integrity verifier, and owned SMPTE LTC timecode generate/read with a burn-in overlay. The streaming reliability & insight release (0.700.0) is about knowing a show will hold up and understanding it afterward: a network-condition simulator for pre-show drills (bandwidth caps, latency/jitter, scheduled outages), a dry-run mode that runs the whole pipeline to a private local sink with zero bytes leaving the machine, a session-long dropped-frame forensic timeline, an optional local stream session report, an encoder benchmark wizard that measures what the machine can sustain, and per-output source visibility (stream-only / recording-only items) — plus looping text effects (a "Star Wars" perspective tilt for any source and a fade-in/out loop) for unattended promos. Everything ships off by default and adds no network call.

Windows / macOS / Linux

  1. Download your platform's installer from the Releases page (Windows .msi/.exe, macOS .dmg, Linux AppImage/.deb/.rpm/Flatpak) and install it. The Windows build is a GUI app with no console window.
  2. Launch Freally Capture. It opens to a clean studio — a program preview, and Scenes, Sources, Mixer, and Controls docks. No account, no login.
  3. On macOS, grant Screen Recording (and Camera/Microphone if you'll use them) in System Settings → Privacy & Security, then relaunch. On Linux (Wayland), capture goes through the desktop portal — you'll pick the source to share.
  4. Add your first source (see "Build a scene") and you're ready to record or go live.

Build a scene

A scene is a stack of sources composed on the GPU. Sources can be shared across scenes.

  1. In the Sources rail, click Add Source and pick a type: Display, Window, Game, Webcam / Capture Card, Browser, Media, Image, Text, Color, or Audio.
  2. Add a Display Capture — the program preview fills with your screen.
  3. Add a Webcam source, then drag it on the canvas to a corner and resize it with the handles.
  4. Add an Image or Browser overlay on top. Reorder sources in the rail to set their z-order (top of the list = front).
  5. Create more scenes in the Scenes rail (e.g. "Starting Soon", "Gameplay", "BRB") and switch between them.
Tip: A source added to one scene can be reused in others — change its properties once and it updates everywhere it appears.
Real-time preview: on Windows, the program preview is painted straight from the GPU compositor ("OBS feel") — drag, scale, and rotate feel instant, with the selection box and handles drawn right into the GPU frame. macOS and Linux use the same on-canvas editing over the standard preview while the native path lands there next.

Filters & transitions

Stack per-source filters on the GPU, and choose how scenes cut to each other.

Add a chroma key

  1. Right-click a source (e.g. a green-screen webcam or overlay) and choose Filters.
  2. Add Chroma Key. Adjust key color, similarity, smoothness, and spill — the keyed color vanishes live on the canvas.
  3. Add more filters as needed (Color Correction, LUT, Blur, Mask, Sharpen). Drag to reorder the chain.

Set a transition

  1. In Controls, open the transition picker and choose Cut, Fade, Slide, Swipe, Stinger, or Luma Wipe, and set a duration.
  2. For a Stinger, pick the media and the transition point; for a Luma Wipe, pick the gradient and softness.
  3. Switch scenes to see the transition play. In Studio Mode, the transition runs when you commit Preview → Program.

Audio mixer

Mix every audio source, route to up to 6 tracks, and clean up your mic — all live.

  1. Each audio source appears as a channel strip in the Mixer — name, VU meter, fader, mute, monitor, filters, and track dots (1–6).
  2. Drag a fader to set level; click mute or monitor as needed; assign the source to any subset of the 6 tracks.
  3. Open a source's filters and add Denoise (classic DSP — no ML), Noise Gate, Compressor, Limiter, EQ, or Gain.
  4. Bind a push-to-talk or push-to-mute hotkey; watch the LUFS meter for program loudness.
macOS note: macOS doesn't hand apps "what you hear" on its own. Use ScreenCaptureKit audio (macOS 13+) where available, or install a virtual device such as BlackHole and select it as your system-audio source. Freally Capture guides you and keeps recording even if a source can't open.

Record

Record your program feed multi-track with your GPU's hardware encoder, or fully lossless in the owned codec.

  1. In Settings → Encoder / Recording, pick an encoder. Freally Capture detects NVENC / Quick Sync / AMF / VAAPI / VideoToolbox and selects the best; x264 (CPU) is always available as a fallback.
  2. Choose a container — mp4 / mkv / mov / webm — or the owned freally-video lossless format (no external tool needed).
  3. Assign sources to audio tracks (up to 6) so you can separate mic, system, and guests in post. Optionally enable file splitting and record a separate-track local copy while streaming.
  4. Click Start Recording in Controls. A green REC dot shows the duration and track count. Click Stop to finalize the file.
Codec note: mp4 with H.264/AAC (and HEVC/AV1) uses patent-encumbered codecs provided by ffmpeg, which is not bundled — Freally Capture fetches it on demand and verifies it the first time you need it. The owned freally-video lossless path needs nothing fetched.

Go live

Broadcast direct to a platform over RTMP/RTMPS — your stream goes straight from your machine, never through a server we run.

  1. Open Settings → Stream and click Add target. Pick Twitch, YouTube, Kick, Facebook, Trovo, or Custom RTMP.
  2. Paste your stream key (stored locally, masked) and set the bitrate/encoder for that service.
  3. Click Go Live in Controls. The LIVE indicator glows and the stats dock shows your bitrate, dropped frames, and stream health.
  4. If your connection blips, Freally Capture auto-reconnects. Your local recording keeps running regardless of stream state. Set a stream delay if you want one.
  5. Click End Stream to stop.

Broadcast safety

The 0.98.0 batch is about not losing the show — before, during, and after.

Before: the go-live pre-flight

Clicking Go Live first runs the checklist: stream targets keyed, a usable encoder detected, every source healthy, and enough disk for the recording — each green/red with a one-click fix — plus honest nudges for microphone/desktop-audio metering and the replay buffer (those never block a screen-only show). An optional setting holds Go Live until every blocking check is green; otherwise "Go Live anyway" is available.

During: alarms, health, and failover

  1. Safety alarms watch while output runs: silent program audio, sustained clipping, a black or frozen program picture, and a low-disk forecast ("~22 min left at the current bitrate"). They appear as a dismissible banner (and are announced to screen readers) — never a modal, never an interruption to the pipeline.
  2. The source health dashboard (palette → "Source health…") lists every source's state, fps, last-frame age, dropped frames, and restart history, with per-source Restart and Properties.
  3. If the hardware encoder faults mid-stream or mid-recording (a driver reset, GPU contention), the encoder failover hot-swaps to the other hardware family or to x264. The stream reconnects and stays up; the recording continues in a new file; a toast + a sticky stats-dock note say exactly what happened. Network faults never trigger a swap.
  4. The panic button (a bindable global hotkey) instantly cuts program — and the vertical canvas and any projectors — to a configurable privacy slate, stops every capture, and hard-mutes all audio (the microphone stops being captured at all) while the stream and recording stay connected. Restoring is a deliberate two-step confirm. Configure the slate colour/image beside the hotkey in Settings → Hotkeys.

After: crash-safe recordings and an orderly quit

  1. mp4/mov recordings are written fragmented, so a crash or power cut leaves a playable file (mkv was already crash-tolerant; the owned .frec always was). If a session ends uncleanly mid-recording, the next launch offers a one-click repair into a (repaired) copy — the original is never touched.
  2. Quitting while live, recording, or replay-armed shows exactly what will happen, then performs an ordered shutdown: end the stream → finalize the recording → flush the replay buffer → save.
  3. Recording filename templates (Settings → Output): token-based names — {prefix} {date} {time} {scene} {profile} {canvas} {marker-count} {counter} — for recordings, replay saves and stills, with per-output folders for replays and stills.
  4. Export diagnostics (the bug-report dialog): a zip of the config snapshot, encoder/device probe, recent stats and the last crash report — built by a strict allowlist, so stream keys, passwords, file paths and names are never included. You can read the exact content first; nothing is ever sent anywhere.

Sources & devices

The 0.99.0 batch — everything between the glass and the canvas: test signals, sync calibration, data-driven text, timers, the hotkey map, mixer console depth, and real device controls.

Test signals & A/V sync calibration

  1. Add a Test Signal source (Sources → + → Test Signal): SMPTE-style color bars, a calibration grid, a motion sweep, the 1 kHz lineup tone (−20 dBFS), or the combined A/V sync flash+beep pattern — video and audio generated from one clock. Verify a projector, an encoder, or a stream target with no camera plugged in.
  2. Open the A/V sync calibration workbench (palette → "A/V sync calibration…"): pick the camera and microphone to align, point the camera at the flashing program (a fullscreen projector helps — the dialog opens one), and let it measure for ~12 seconds. It detects the flash and beep onsets over several cycles and offers to apply the measured offset to the microphone's sync offset; the manual slider remains the fallback. Honest note: the loop runs through your display and speakers, so their own small latencies are part of what's measured — and failures say why (camera never saw the flash, mic never heard the beep, room events that aren't the pattern).

Timers, clocks, and text from a file

  1. A Timer source shows a wall clock (strftime format, optional fixed UTC offset), a countdown (to a duration or the next wall time — at zero it can flash or switch scene), a stopwatch, or time since live / since recording. Start/pause/reset from the source's Properties, or bind the two global hotkeys that drive all timers at once (Settings → Hotkeys).
  2. A Text source can read from a watched file (Properties → Read from file): the whole .txt, one CSV cell (by header name or index), or one JSON value (an RFC 6901 pointer like /teams/0/score). Changes appear within half a second; atomic-write editors (temp + rename) are tolerated by holding the last good value. Scoreboards and lower thirds with no service.

The hotkey map & the mixer console

  1. The hotkey map (palette → "Hotkey map…") lists every binding — global actions and per-source push-to-talk/push-to-mute — searchable and filterable, with honest conflict signals: the same chord written two ways is flagged as shared, and a key the OS refused to register (owned by another app, or unavailable on this compositor) says so. Export cheat sheet writes a Markdown reference into Downloads.
  2. Every mixer strip now has pan (stereo balance — centered is bit-identical to no pan; double-click resets), PFL solo (the monitor hears only soloed strips; the program mix never changes), and a mono downmix — pan and mono live in the strip's "⋯" popover, solo on the strip face.

Device depth: deinterlacing & camera controls

  1. A Video Capture Device's Properties now offer deinterlacing for interlaced feeds (capture cards fed by older cameras/consoles): discard, bob, linear, blend, or motion-adaptive, with field-order selection. Pure CPU on the capture thread — identical on every OS; changing the mode restarts the device, like a format change.
  2. Camera controls (exposure, white balance, focus, zoom, gain, and more) appear in the same Properties while the device is streaming. Windows and Linux use the native UVC paths; macOS surfaces whatever AVFoundation reports — often nothing, and the panel says so rather than showing fake sliders. Every adjustment saves into a per-device profile that reapplies automatically on hotplug and restart; Reset profile returns the backend defaults.

Multistream

Stream to several platforms at the same time — direct from your machine, no restream server, completely free.

  1. Add two or more stream targets in Settings → Stream and enable each.
  2. Set per-service encoder settings if a platform needs a different bitrate.
  3. Click Go Live — you broadcast to all enabled targets at once. The stats dock shows each target's health and bitrate.
  4. Optionally record a separate-track local copy at the same time, and save replays with the replay hotkey while you stream.
Free for everyone: every feature — multistream, SRT/WHIP, advanced filters/scripting, replay presets, the stinger packs, and remote-control automation — is free. No paid tier, no payments, no ads.

Virtual camera

Turn your produced scene into a webcam for any video-call app.

  1. Click Start Virtual Camera in Controls. Freally Capture registers a system camera named "Freally Capture Virtual Camera".
  2. In Zoom, Meet, Discord, or Teams, choose that camera in the video settings — your composed program feed becomes the webcam.
  3. Click Stop Virtual Camera to remove the device.
Linux note: the virtual camera uses v4l2loopback, a kernel module you install once. Freally Capture guides you through it and never installs system components silently. macOS uses a bundled, notarized CoreMediaIO plugin.

Replay buffer

Keep a rolling buffer so a hotkey saves the moment that just happened.

  1. Click Start Replay Buffer in Controls. Freally Capture keeps the last N seconds in a bounded buffer.
  2. Bind a global Save Replay hotkey in Settings.
  3. Press the hotkey to save the last N seconds to a file — without interrupting your stream or main recording. A "Replay saved" toast confirms it.

Remote control

Drive the studio from a Stream Deck or Companion-style controller — off by default, password-protected, on your machine.

  1. Open Settings → Remote Control and enable the WebSocket API (off by default). Set a password.
  2. It binds to 127.0.0.1 by default; LAN access is an explicit opt-in.
  3. Point a Companion / Stream Deck client at the shown WebSocket URL. You can switch scenes, commit transitions, toggle mutes, start/stop recording, and save replays from hardware buttons.

Settings

Profiles, scene collections, hotkeys, and language — everything is free, so there is nothing to activate. Since 0.301.0, settings live in a categorised modal (General · Appearance · Streaming · Output · Replay · Hotkeys · Network · Accessibility · About) reached from the menu bar — File · Edit · View · Docks · Profile · Scene Collection · Tools · Help across the top of the window.

  1. Apply means apply. The Settings modal edits a private copy — nothing changes until you click Apply (or OK). Cancel restores exactly what you had, live theme preview included.
  2. Profiles & scene collections: keep separate setups (e.g. "Streaming" vs "Local recording") and switch them without restarting, from the Profile and Scene Collection menus.
  3. Hotkeys: bind every action from a conflict-proof dropdown — pick a shortcut for one action and it disappears from every other action's menu, so two features can never share a key; "None" frees it again.
  4. Accessibility: set the mixer meter colours, including a colour-blind-safe preset.
  5. Language: pick from 18 languages (English first); the UI updates immediately, with full RTL for Arabic.

Privacy

Freally Capture is local-first and account-free. Here is exactly what it does and does not do with your data.

  1. No account, no cloud, no telemetry. The app never requires a login and never sends your scenes, recordings, or any usage data to any server.
  2. Streams go direct. Your broadcast goes from your machine straight to the platforms you configured — there is no restream server we run.
  3. Recordings stay on your device. They are written to the folder you choose. Nothing is uploaded.
  4. Stream keys are local. They are stored on your machine, masked in the UI, and sent only to the service you stream to.
  5. Nothing bundled but the engine. ffmpeg (for the wire codecs the platforms require) and the optional virtual-background model are not bundled — they're fetched on demand and verified, and the app shows you what it will fetch.
  6. For the full policy, see PRIVACY.md on GitHub.
